# 总部学员

> 来源: https://www.yuque.com/schoolpal-wiki/manual/izofhde72wpbibln

场景释义

多校区的机构，在总部学员列表，可以统一查看机构所有校区的学员详情，包括是否报名、报名校区、报名课程等，并且可以为学员办理跨校区转课或上课业务。

操作步骤

总部学员管理

总部学员分类查看

**路径：**总部管理后台 >> 跨校办理 >> 总部学员 >> 分类查看总部学员详情

注：支持查看机构学员总数，意向学员、在读学员、历史学员，以及总部未分配学员。
意向学员：有分配校区，且任一校区都未报名过正价课程的总部学员
在读学员：有分配校区，且任一校区是在读状态的总部学员
历史学员：有分配校区，且没有校区在读，有任一校区是历史学员状态的总部学员
未分配：暂未分配校区的总部学员

总部学员和校区学员的关联关系

**路径：**总部管理后台 >> 跨校办理 >> 总部学员 >> 学员详情 >> 分配校区或解除关联

注：一位总部学员关联的多位校区学员，可以查看各校区的报读或跟进详情，可以分配校区，可以解除关联（比如意向学员 A 分配给校区二后，需要更换分配校区，可以再分配到校区一，然后解除校区二的关联）
一位总部学员在一个校区只能关联一位学员，当一个校区已经有学员和总部学员 A 关联，校区创建第二个满足关联条件的学员时（比如手机号和 A 相同），不会关联总部学员 A，而是单独创建一个新的总部学员

新增总部学员

按照设置的校区管控规则，校区在创建学员时，会关联到总部已有的学员，或在总部生成一个新的学员信息进行关联

批量导入总部学员

**路径：**总部管理后台 >> 跨校办理 >> 总部学员 >> 导入 >> 批量新增总部学员

注：一个总部学员关联的多个校区学员，可以查看校区的报读或跟进详情，可以分配校区，可以解除关联（比如意向学员 A 分配给校区二一段时间后，需要更换分配校区，可以再分配到校区一，然后接触校区二的关联）
多个总部学员（姓名和手机号可能相同），可以选择合并为同一学员，再统一办理跨校相关业务

常见问题

原有的跨校学员怎么看？

在总部学员列表中，筛选“有跨校上课校区”的学员，即为原来的跨校学员

总部学员是什么，和各个校区的学员有什么区别？

- 总部学员就是各个校区的学员档案，在总部按姓名+手机号（或手机号）相同，汇总为同一个人，便于查看这个人在不同校区的报名和上课情况进行统一管理
- 单校区创建学员，也会自动去按设置的条件找到总部学员进行关联；找不到也会创建一个新的总部学员进行关联
- 如果想要调整关联关系，可以手动解除关联，或者合并总部学员
- 总部学员任何关联与合并，都不影响单校区学员档案的数据信息
- 在总部学员列表，筛选小区后，和对应校区的学员数据是一致的：比如，总部学员列表中筛选校区一在读学员有 100 个，校区一在单校的学员管理中，筛选在读学员也是 100 个

跨校上课办理，办不了怎么办？

- 目标校区没有可跨校上课的课程，需要先检查两个校区的课程是否支持跨校上课，且办理跨校上课业务的学员是否在原校区已报名此课程
- 目标校区已存在相同姓名+手机号学员，但与原校区学员不在同一个总部学员关联下，可以先检查总部学员的关联情况，做合并，或解绑后再合并，满足条件后再进行办理跨校上课业务

导入更新学员

更新学员信息时，如果总部没有对校区的学员自定义属性和渠道做过统管设置，则总部的学员属性不显示；

需要总部先设置好对应的统管设置，再更新学员信息，即可显示学员的属性或渠道信息
