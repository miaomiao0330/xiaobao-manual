# 课程管理

> 来源: https://www.yuque.com/schoolpal-wiki/manual/ouv748pn8fmvrw0e

一、场景释义

多校区机构进行运营管理时，对于整体业务而言，课程商品为首要管控目标。对所有校区管理的前提，优先是保证所有校区售卖、教学的课程具有统一性。

多校区课程管理通过建立“总部创建课程模板统一分发”的模式，实现对所有校区课程可控可查可统计：

对于直营类型的校区，课程一般都是来自于总部，并且随总部课程变化而更新；

对于加盟类型的校区，课程有部分来自总部，部分则是由加盟校自研。

三、常见问题

为什么分发课程时，提示分发同步失败？

因为在向分校分发同步时，检测到分校不满足同步条件。例如：总部在分发课程时，分校暂未设置课程中包含的课程属性，则该课程就会分发失败

导致课程分发失败的原因有哪些？该如何处理？

| **失败原因** | **处理方法** |
| --- | --- |
| 校区未启用 1v1 课程配置 | 前往校设置统管 >> 教务设置 >> 课程设置 >> 总部设置 >> 授课方式，开启 1v1 授课，并同步给校区 |
| 校区未启用通用课程配置 | 前往校设置统管 >> 教务设置 >> 课程设置 >> 总部设置 >> 授课方式，开启通用课，并同步给校区 |
| 校区未启用按金额收费配置 | 前往校设置统管 >> 教务设置 >> 课程设置 >> 总部设置 >> 收费方式，开启按金额收费，并同步给校区 |
| 校区未启用按时段收费配置 | 前往校设置统管 >> 教务设置 >> 课程设置 >> 总部设置 >> 收费方式，开启按时段收费，并同步给校区 |
| 校区课程商品名称重复 | 修改总部课程名称或修改校区已有的课程名称 |
| 校区科目不匹配 | 前往校设置统管 >> 教务设置 >> 课程设置 >> 校区设置，开启课程设置同步给校区，或前往校区新建课程科目 |
| 校区课程属性不匹配 | 前往校设置统管 >> 教务设置 >> 课程设置 >> 校区设置，开启课程设置同步给校区，或前往校区新建课程属性 |
| 校区课程类别不匹配 | 前往校设置统管 >> 教务设置 >> 课程类别 >> 校区设置，开启课程类别同步给校区，或前往校区新建课程类别 |
| 校区不存在已选允许老生购买的课程 | 修改该课程中，微校课程设置允许老生购买的课程范围或先将已选允许老生购买的课程分发给对应校区后，再分发此课程 |
| 部分班课通用已选课程范围与校区课程不匹配 | 修改课程中的课程范围或将已选课程范围中的课程分发给校区后，再分发此课程 |

为什么我的总部课程属性、课程类别都是空的？

为了避免总部管理员在配置时因误操作（如不小心勾选所有分校）导致将未完善的属性/类别同步到所有分校，系统默认总部的属性/类别是空的。您需要：

- 开启需要同步的分校的同步开关。
- 在总部创建所需的课程属性和类别。

注意： 开启分校同步开关和创建总部属性的先后顺序没有强制要求。无论先做哪一步，只要分校同步开关是开启状态，您在总部创建新属性/类别后，它们都会自动同步到已开启开关的分校。

为什么之前可以在总部管理校区课程，现在菜单不见了？

这是由于系统权限策略调整。您当前的账号缺少了查看和操作分校课程的权限。请联系拥有总部管理员权限的同事，为您在权限配置中开启相应的 “课程管理”权限。

分校之前自己创建的课程属性、课程类别会受影响吗？

以前分校创建的课程属性、类别不会受影响，分校仍然可以正常使用，如果要批量新增课程属性或类别，建议通过多校管理后台进行创建后，开启自动同步，避免重复操作，提供效率。
